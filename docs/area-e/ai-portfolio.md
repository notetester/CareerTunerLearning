# 포트폴리오 설명 첨삭 [#27]

> `correctionType=PORTFOLIO`로 프로젝트의 배경, 본인 역할, 문제 해결과 결과를 채용자가 읽기 좋은 설명으로 다듬는다.

## 무엇이 다른가

포트폴리오도 공통 correction 서비스와 모델을 사용한다. 별도점은 prompt가 다음 구조를 강조한다는 것이다.

- 프로젝트 배경과 목표
- 본인 역할과 팀 역할 구분
- 기술 선택 이유
- 해결한 문제와 검증 근거
- 결과와 배운 점

현재 입력은 텍스트와 선택적 지원 건·질문이다. 외부 포트폴리오 URL 크롤링이나 PDF/DOCX 파싱을 구현했다고 과장하지 않는다.

## 사실 보존

없는 트래픽, 매출, 성능 향상, 사용자 수를 만들지 않는다. 추가하면 좋은 근거는 `suggestions`에 남기고 개선문과 변경 이유를 분리한다. 재첨삭은 새 행으로 남고 삭제는 소프트 삭제다.

## provider와 과금

AUTO는 CareerTuner 자체 LoRA → Claude → OpenAI다. 모델 선택, preflight, payload 검증, 사용량 로그, 실제 차감과 요청 키 멱등 계약은 다른 첨삭 유형과 같다. 전 provider 실패 시 Mock 개선문을 저장하지 않는다.

## 화면

`Correction.tsx` 포트폴리오 탭은 실제 API를 호출한다. 입력, 지원 건, 모델, 예상 비용, 실행 결과와 최근 이력을 제공하며 반응형 배치에서 작은 화면은 한 열로 정리된다.
