# 왜 아는데 말이 막히나 — 학습/말하기 전략

> "알긴 아는데 입으로 안 나와요"는 지식이 부족한 게 아니라, **꺼내는 연습**을 안 한 것이다. 이 사이트는 그 꺼내기 근육을 만든다.

## 1. 한 줄 정의

면접에서 막히는 진짜 이유는 "이해(읽으면 안다)"와 "인출(아무것도 안 보고 말로 꺼낸다)"이 **별개의 능력**이기 때문이고, 이 페이지는 인출을 훈련하는 8단계 사이클과 이 사이트 사용법을 알려준다.

## 2. 단어 뜻 (왜 막히는지의 정체)

| 용어 | 뜻 | 면접에서 |
| --- | --- | --- |
| 이해(Comprehension) | 자료를 보면서 "맞아, 이거지" 하고 따라가는 능력 | 코드를 읽으면 다 안다 |
| 인출(Retrieval) | 단서 없이 머릿속에서 끄집어내는 능력 | 면접관 앞에선 자료가 없다 |
| 유창성 착각(Fluency Illusion) | 술술 읽힌다고 "안다"고 착각하는 현상 | 문서를 여러 번 읽고 안다고 착각 |
| 능동 회상(Active Recall) | 일부러 떠올려보며 외우는 방법 | 인출을 직접 연습하는 것 |

:::tip 핵심 한 줄
**읽기는 인풋, 면접은 아웃풋이다.** 인풋만 백 번 해도 아웃풋은 안 는다. 그래서 이 사이트의 모든 페이지는 끝에 퀴즈(인출 연습)가 붙어 있다.
:::

## 3. 왜 필요한가 (없으면 무슨 문제가 생기나)

말문이 막히는 전형적 3패턴과 원인:

- **첫 문장이 안 나온다** → 정의를 외운 적이 없다. "JWT가 뭐예요?"에 한 문장으로 답할 준비가 안 됨.
- **개념은 아는데 우리 프로젝트랑 연결이 안 된다** → "그래서 너는 어디에 썼냐"에서 무너진다. CareerTuner에서 `JwtTokenProvider`, `ApiResponse`, `FitAnalysisAiService` 같은 실제 위치를 못 댄다.
- **꼬리질문에서 무너진다** → 첫 답은 외웠는데 "왜 그렇게 했냐"는 후속에 대비를 안 했다.

이 세 구멍을 막는 게 아래 8단계다. 단계마다 한 패턴씩 정확히 겨냥한다.

## 4. 8단계 학습 사이클 (이 사이트의 페이지 구조 그대로)

기술/개념 페이지는 전부 이 순서로 쓰여 있다. **읽는 순서가 곧 외우는 순서**다.

| 단계 | 무엇을 | 막히는 패턴 중 무엇을 막나 |
| --- | --- | --- |
| 1. 단어 뜻 | 약자/어원을 풀어 말로 | "이름은 들어봤는데"를 없앤다 |
| 2. 개념 | 한 줄 정의 | 첫 문장이 안 나오는 문제 |
| 3. 왜 | 없으면 생기는 문제 | "그게 왜 필요해요?" 대응 |
| 4. 내 코드 위치 | 실제 클래스/파일/테이블 | "어디에 썼냐"에서 안 무너지게 |
| 5. 원리 | 표/작은 코드/단계 | 깊이 질문 대응 |
| 6. 면접 답변 3단계 | 1문장 / 기본 / 꼬리 | 실전 스크립트 확보 |
| 7. 꼬리질문 | 후속 + 모범답안 | 후속 공격 방어 |
| 8. 직접 말해보기 | 소리 내서 답하기 | 인출을 실제로 실행 |

:::warning 1~5만 하고 멈추면 똑같이 막힌다
1~5는 전부 **인풋**이다. 면접이 막히는 사람의 99%가 여기까지만 한다. 6~8(아웃풋)을 안 하면 읽기 전과 똑같이 입이 안 떨어진다. **8단계는 옵션이 아니라 핵심이다.**
:::

## 5. 핵심 동작 원리 (왜 이 순서가 먹히나)

### 정의 → 위치 → 원리 → 후속, 면접 답변의 자연 순서

면접 답변은 **좁게 시작해 넓혀가는** 깔때기다. 8단계는 이 깔때기를 그대로 따른다.

```text
[1문장 정의]  →  [우리 프로젝트 어디에]  →  [어떻게 동작]  →  [왜 그렇게]
   2단계            4단계                   5단계            7단계
```

### 능동 회상: 답을 먼저 가린다

```text
나쁜 공부:  페이지 정독 → "이해됨" → 다음 페이지   (유창성 착각)
좋은 공부:  질문만 보기 → 입으로 답해보기 → 그다음 본문으로 확인
```

퀴즈를 **풀고 나서** 해설을 보는 이유가 이것이다. 틀려도 좋다. 떠올리려고 애쓴 그 순간에 기억이 박힌다.

### 30초 답변을 미리 조립해 둔다

| 시간 | 구성 | 예시(JWT) |
| --- | --- | --- |
| 0~5초 | 한 줄 정의 | "JWT는 서버가 서명한 자가검증 토큰입니다." |
| 5~20초 | 우리 프로젝트 위치 | "CareerTuner에선 `JwtTokenProvider`로 발급하고 `JwtAuthenticationFilter`가 Bearer를 파싱합니다." |
| 20~30초 | 핵심 선택 1개 | "Access 30분 / Refresh 14일 DB저장으로 탈취 대응을 했습니다." |

이 30초가 머릿속에 통째로 들어 있으면, 어떤 기술 질문이 와도 첫 30초는 무너지지 않는다.

## 6. 이 사이트 사용법 3단계 (실전 루틴)

**초간단(매일 5분):** 아무 페이지나 열고 6단계의 "1문장"만 소리 내어 말한다. 막히면 그 페이지가 오늘 공부할 페이지다.

**기본(한 페이지 정복):**
1. 페이지를 1~5단계까지 읽는다(인풋).
2. 스크롤을 6단계로 내려 **본문을 가리고** 1문장 → 기본 → 꼬리를 입으로 말한다.
3. 7단계 꼬리질문을 보고, 답을 먼저 말한 뒤 모범답안과 비교한다.
4. 맨 아래 퀴즈를 **먼저 풀고** 해설을 본다.

**꼬리(면접 직전):** 7단계 꼬리질문만 쭉 훑으며 답이 바로 나오는지 점검. 막히는 것만 표시해 다음날 다시.

:::tip 연결 학습
한 개념이 다른 개념을 부른다. 본문 안의 링크(예: [DTO](/glossary/dto), [JWT](/backend/jwt-security), [ApiResponse 엔벨로프](/glossary/api-response-envelope))를 타고 가며 묶어서 외우면 꼬리질문에 강해진다. 4단계(내 코드 위치)가 막히면 [기술 지도](/guide/tech-map)로 가서 기능→파일을 한 번에 훑어라.
:::

## 7. 자주 나오는 꼬리질문 + 모범답안

::: details "그냥 외우는 거랑 뭐가 달라요?"
외우기는 인풋, 인출은 아웃풋입니다. 능동 회상은 답을 가리고 떠올리는 과정 자체로 기억을 강화하기 때문에, 같은 시간을 써도 면접장에서 꺼내지는 확률이 훨씬 높습니다. 단순 반복 읽기는 유창성 착각만 키웁니다.
:::

::: details "프로젝트에서 안 쓴 기술도 이렇게 공부해야 하나요?"
4단계(내 코드 위치)는 비워두거나 "우리 프로젝트에는 적용하지 않았습니다"라고 정직하게 말하면 됩니다. CareerTuner의 C 자체 모델처럼 학습·연결은 완료했지만 운영 기본값은 외부 provider인 경우도 있으므로, 구현 여부·설정 활성·운영 기본을 나눠 설명하는 것이 오히려 신뢰를 줍니다.
:::

::: details "30초 답변을 외우면 앵무새처럼 들리지 않나요?"
30초는 뼈대일 뿐입니다. 정의·위치·선택 1개라는 골격만 고정하고 살은 그때그때 붙입니다. 골격이 있으면 긴장해도 첫 문장이 나오고, 그 한 문장이 나오면 나머지가 따라옵니다.
:::

::: details "퀴즈를 틀리면 시간 낭비 아닌가요?"
반대입니다. 떠올리려다 실패한 직후에 정답을 보면 기억이 가장 강하게 박힙니다(인출 실패 후 교정). 틀리는 게 손해가 아니라, 안 떠올려보고 해설부터 읽는 게 손해입니다.
:::

::: details "면접 직전 벼락치기로 이걸 쓸 수 있나요?"
직전엔 7단계(꼬리질문)만 빠르게 훑으세요. 답이 입에서 바로 나오면 통과, 막히면 표시. 새 개념을 넣기보다 이미 본 것의 인출 속도를 올리는 게 직전 전략입니다.
:::

## 8. 직접 말해보기

지금 바로, 소리 내서 답해보자. 머릿속으로만 하면 효과가 절반이다.

1. **"왜 읽으면 아는데 면접에선 막히는지"를 30초 안에 후배에게 설명한다고 생각하고 말해보라.** (힌트: 이해 vs 인출, 유창성 착각)
2. **이 사이트의 8단계 중 자신이 평소 건너뛰던 단계 하나를 고르고, 왜 그게 치명적인지 말해보라.**
