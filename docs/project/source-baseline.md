### 직전 기준선 이후 표적 확인

직전 Learning 기준선 `d00a57fc` 이후 변경은 전 영역 전체 재검증이 아니라 변경 파일과 최종 원장의 증거를 따라 확인했다.

- A: 프로필 전체·섹션 저장과 문서 가져오기에 버전 기반 3-way merge, 늦게 도착한 응답 방어, 미저장 이탈 경고와 원본 파일 정리가 추가됐다. 프로필은 `/profile/*` 분리 경로를 실제 사용한다.
- B/C: 지원 건 비교·전략·학습, 취업분석, 플래너, 카탈로그가 허브와 세부 경로로 분리됐고 NCS·자격증 카탈로그 API/DB가 추가됐다.
- D/E: AutoPrep 취소·첨부 재시도·빈 면접 세션 정리와 모델 재시도 선택 계약, PG 승인 전 유료 권한 금지 경계를 보강했다.
- F: 커뮤니티 차단·탈퇴 사용자 필터와 관계 soft delete, 서버 전수 카테고리 집계, 채용공고 전용 게시판 이동 경계를 보강했다.
- 플랫폼/배포: PR #448의 필수 CI, DB patch 적용, backend/web/mock 배포와 Android live release가 통과했다. 데스크톱 소스는 바뀌지 않아 기존 패키지 실기 증거를 승계한다.

운영 OAuth 3종, Claude Haiku/OpenAI GPT, SMS는 프로젝트 책임자의 수동 live 완주(`PASS_MANUAL`)다. Apple은 개발자 활성화·테스트 단계이며 Team ID 기반 iOS Universal Link는 시연 범위 밖 `DEFERRED`다.

## 무엇을 사실로 채택하는가

같은 주제의 자료가 서로 다르면 다음 순서로 판단한다.

1. 실제 요청을 처리하는 컨트롤러·서비스·매퍼·프런트 라우트
2. `schema.sql`과 배포되는 증분 DB 패치
3. 빌드 설정과 패키지 잠금 파일
4. 자동 테스트와 `docs/verification/DEMO_READINESS_LEDGER.md`
5. 모듈 README와 아키텍처 문서
6. 초기 기획, 클래스 설계안, 과거 학습 문서

초기 문서는 목표 상태를 설명할 수 있다. 반대로 코드가 존재한다는 사실만으로 운영 설정·외부 자격증명·실기기 검증까지 완료됐다고 단정할 수도 없다.

## 구현 상태를 쓰는 네 가지 표현

| 표현 | 사용 조건 | 예시 |
| --- | --- | --- |
| 구현·연결됨 | 사용자 경로에서 실제 호출되고 테스트 근거가 있음 | 프로필 버전 조회, 첨삭 이력 삭제 |
| 설정 시 활성 | 코드와 배선은 있으나 런타임 설정이나 공급자가 필요함 | 자체 모델, OAuth, RAG |
| 실험 완료 | 학습·평가 산출물은 있으나 운영 기본 경로가 아님 | 일부 LoRA, C RAG 비교 |
| 근거 미확인 | 이름이나 설정은 있으나 학습·배포 provenance가 부족함 | F `careertuner-mod`의 직접 학습 주장 |

“코드가 있다”와 “운영 기본값이다”, “학습했다”와 “공개 베이스 모델을 자체 호스팅했다”를 섞지 않는 것이 핵심이다.

## 공개 문서에서 제외하는 정보

이 저장소는 포트폴리오 학습 자료이므로 다음은 옮기지 않는다.

- API 키, 토큰, 비밀번호, 인증서 원문
- 개인 이메일, 전화번호, 실사용자 식별자
- 내부 IP, 사설 호스트, 운영 서버 접속 명령
- 로컬 장비의 절대 경로와 개인 환경 파일
- 실제 OAuth·결제 공급자 콘솔의 비밀 설정값

대신 “환경변수로 주입”, “HTTPS 반환 주소 등록 필요”, “자체 호스팅 endpoint가 설정되어야 활성”처럼 구조만 설명한다.

## 문서 갱신 절차

새 PR이 기능을 바꾸면 전체 문서를 다시 훑지 않고 다음 순서로 좁힌다.

1. 기준 SHA 이후 변경 파일을 구한다.
2. 기능별 검증 원장의 매핑으로 영향 영역과 플랫폼을 선택한다.
3. 해당 페이지의 현재 상태·근거 경로·퀴즈 해설만 갱신한다.
4. `npm run docs:check:stale`로 폐기된 문구가 돌아오지 않았는지 확인한다.
5. VitePress 빌드를 실행해 내부 링크를 모두 검증한다.
6. 이 페이지의 SHA와 검토일을 새 기준으로 올린다.
